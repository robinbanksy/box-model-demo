JavaScript Introduction

History

During the early days of the web, all pages were static. This meant that once a page was loaded, it couldn't dynamically change content without a reload of the whole page. In order to remedy this, Netscape, the creators of Netscape Navigator, at the time the most popular GUI web browser, wanted to embed a programming language in their browser. An early idea was to use the popular Java language, but instead, a new language was developed with the goal of being easier for non-programmers to learn and use. The laguage was called JavaScript, partially as a marketing ploy to associate the new language with Java, which was extremely popular at the time, despite having no relation to the Java language.

JS Runtime

Traditional programming languages work in a system of source code and machine code. For example, a program written in the C language is written, and then fed to a program called a compiler, which translates the program into binary machine code, which can be natively run on the targeted hardware.

JavaScript is different. It is not compiled into machine code, so it requires a special "runtime environment" to execute the program. Originally, since JavaScript was designed for web browsers, this runtime environment was embedded in the browser. As a result, JavaScript could only run in-browser. Since then, the runtime environment has been extended to other platforms, such as Node.js, which allows JavaScript to be run on the server-side (i.e., outside of the browser).

Node.js

Node.js is a server-side JavaScript runtime environment. Think of it as the same JavaScript engine originally meant for the browser, but taken out of the browser environment.

In order to use Node, you'll need to install it: https://nodejs.org/en/download

Once you've got Node installed, you can start a Node.js server by running the command node in your terminal.

Programming Intro

Within our Node environment, we can execute single lines of JavaScript code directly in the terminal. Try entering the following lines and note their results:

2+2
4
2>3
console.log("Hello, world!")

Of course, if we want to do more interesting things, we're going to have to write more complex programs. We'll do this by creating a JavaScript file, with the .js extension.

Variables

Variables are used to store data in JavaScript. You can create a variable using the let keyword.

let x = 5;

You can then use the variable in your code:

console.log(x);

Once a variable is declared, you can change its value:

x = 10;
console.log(x);

Notice that we don't use the let keyword again when reassigning a value to an existing variable. That would effectively be declaring the variable again, which would result in a syntax error.

Similar to let, you can also use the const keyword to declare a variable that cannot be reassigned:

const y = 10;
console.log(y);

Attempting to reassign a value to a const variable will result in a syntax error.

y = 15; \\ This will cause a syntax error

Data Types

Data types are the different categories of values that variables can hold in JavaScript. We've already seen a few data types in action:

Numbers: Any numbers, including integers and floating-point numbers

let num = 5;

Strings: Text values enclosed in quotes

let str = "Hello, world!";

Booleans: Logical values that are either true or false

let bool = true;

Conditional Statements

Conditional statements allow you to execute different blocks of code based on certain conditions.

if (x > 5) {
    console.log("x is greater than 5");
}

In the above code, the console.log statement will only execute if the condition (x > 5) is true. Notice that the condition is specified in the form of a boolean expression; in other words, it evaluates to either true or false.

You can also add an else block to execute code when the condition is false:

if (x > 5) {
    console.log("x is greater than 5");
} else {
    console.log("x is not greater than 5");
}

In this case, if the condition (x > 5) is false, the code in the else block will execute instead.

Even more complex conditional logic can be achieved using else if statements:

if (x > 5) {
    console.log("x is greater than 5");
} else if (x === 5) {
    console.log("x is equal to 5");
} else {
    console.log("x is less than 5");
}

In this case, the code will check the first condition (x > 5) and execute the corresponding block if it's true. If it's false, it will check the second condition (x === 5) and execute that block if it's true. If both conditions are false, it will execute the else block.

We could string together multiple else if statements to create even more complex conditional logic.

Functions

Functions are blocks of code that can be defined once and called multiple times throughout your program.

function greet(name) {
    console.log("Hello, " + name + "!");
}

In the above code, we define a function named greet that takes one parameter, name. When the function is called, it will execute the code inside the function body.

greet("Alice"); // Output: Hello, Alice!
greet("Bob");   // Output: Hello, Bob!

Functions can also return values using the return keyword:

function add(a, b) {
    return a + b;
}

In this case, the add function takes two parameters, a and b, and returns their sum. We can call the function and store its return value in a variable:

let sum = add(3, 4);
console.log(sum); // Output: 7
