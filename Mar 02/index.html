Responsive Web Design

Responsive web design (RWD) ensures that your website is usable on all devices, from desktops to smartphones. This is especially important today, since a majority of internet traffic now comes from mobile devices.

Since every website is different, RWD is more a set of guidelines and best practices than a rigid set of rules. My advice would be the follow certain steps when thinking about how to lay out your pages:

  1. The first step is realizing that HTML pages actually tend to be responsive by default! Often issues are caused by CSS that we write that overrides the default behavior. So when writing CSS, be mindful of how you do it
  2. Use flexible grid layouts that can adapt to different screen sizes. This often involves using CSS Grid or Flexbox.
  3. Make images and media flexible so they can scale within their containers.
  4. Use media queries as a last resort, when other responsive techniques are not sufficient. For example, if you have a very complex layout that doesn't adapt well to smaller screens, you might use media queries provide an alternative layout for mobile devices.

Images

Images are one of the few cases where default HTML behavior can fail us. A very large image can cause the dreaded side-scroll.

Thankfully, the fix is usually fairly straightforward: set the image's max-width to a percentage and set its height to auto. This ensures that the image scales down to fit within its container while maintaining its aspect ratio.

Media Queries

Media queries allow you to apply different styles based on the device's characteristics, such as screen width.

These queries are very powerful for when we want one set of styles for small screens and another for larger screens, for example. However, the best practice is still to focus more on making sure the layout works for all devices by default.
