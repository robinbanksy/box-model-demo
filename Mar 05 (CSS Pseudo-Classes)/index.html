CSS Pseudo-Classes

Pseudo-classes are keywords added to selectors that specify a special state of the selected elements. They allow us to apply styles based on user interaction, element position, or other conditions without needing to add extra classes or IDs to your HTML.

Pseudo-classes are defined using a colon (:) followed by the pseudo-class name. For example, :hover is a pseudo-class that applies styles when the user hovers the mouse over an element.

Hover Pseudo-Class

The :hover pseudo-class applies styles when the user hovers the mouse over an element.

For example, you can change the background color of a button when it's hovered:

button:hover {
    background-color: var(--primary-color);
}

This will make the button's background color change when the user hovers over it. Any styling corresponding to the hover selector will override the default styles. This means we don't need to stop at just changing the color. Pretty much any CSS property can be changed on hover.

Some more examples:

button:hover {
    background-color: var(--secondary-color);
    color: white;
    border-radius: 25px;
}
button:hover {
    transform: scale(105%);
    background-color: var(--primary-color);        
}

Focus Pseudo-Class

The :focus pseudo-class applies styles when an element has received focus, either through keyboard navigation or by being clicked on.

For example, you can change the border color of an input field when it receives focus:

input:focus {
    border-color: var(--primary-color);
    outline: none;
}

This will change the outline color of the input field when it receives focus. The outline: none; property is often used to remove the default focus outline, but be cautious when doing this, as it can affect accessibility. Always ensure that there is some visual indication of focus for users who rely on keyboard navigation.

Just like with :hover, :focus can be used to override CSS properties when an element is in the focus state.

Focus-Within Pseudo-Class

The :focus-within pseudo-class applies styles to an element when it or any of its descendants have received focus.

For example, you can change the background color of a form when any of its input fields receive focus:

form:focus-within {
    background-color: var(--primary-color);
}

This is similar to the :focus pseudo-class, but it applies to the parent element when any of its descendants have focus. This is useful for highlighting a form that the user is currently interacting with.

Enabled and Disabled Pseudo-Classes

The :enabled pseudo-class applies styles to elements that are enabled, while the :disabled pseudo-class applies styles to elements that are disabled.

For example, you can style enabled and disabled buttons differently:

button:enabled {
    background-color: var(--primary-color);
    color: white;
}
button:disabled {
    background-color: var(--secondary-color);
    color: gray;
}

In this example, the enabled button will have a primary background color and white text, while the disabled button will have a secondary background color and gray text. This allows users to easily distinguish between interactive and non-interactive elements on the page.

Link and Visited Pseudo-Classes

The :link pseudo-class applies styles to links that have not yet been visited, while the :visited pseudo-class applies styles to links that have already been visited.

For example, you can style unvisited and visited links differently:

a:link {
    color: var(--primary-color);
}
a:visited {
    color: var(--secondary-color);
}

In this example, the link will be styled with the primary color when it has not been visited, and it will change to the secondary color once it has been visited.

nth-child Pseudo-Class

The :nth-child(n) pseudo-class applies styles to elements based on their position within a parent element. The n can be a number, a keyword, or a formula.

For example, you can style every even child of a list differently:

li:nth-child(even) {
    background-color: var(--primary-color);
    color: white;
}
