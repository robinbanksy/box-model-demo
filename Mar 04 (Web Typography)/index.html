Web Typography.

Typography is the art of arranging type in order to ensure that it is readable and aesthetically pleasing. Web typography, in kind, is the practice of using typography on the web.

Today we'll be exploring the principles of web typography, and some practical tips on how to handle fonts for your webpages.

Principles of Web Typography

  • Readability: Your text should be easy to read. This includes choosing the specific fonts you're going to use, but also involves setting proper font sizes, line heights, and letter spacings.
  • Hierarchy: Use different font sizes and weights to create a visual hierarchy that guides the reader's eye through the content.
  • Consistency: Maintain a consistent typographic style throughout your website to create a cohesive look.

Font Selection

Finally, let's talk about actually choosing fonts. There are a number of considerations when doing so:

  • Web Safety: Choose fonts that are widely supported across different browsers and devices. The rest doesn't matter if your font can't be displayed at all. If you want to use an uncommon or custom font, make sure to ship it with your website.
  • Style: There are a few different styles of fonts to consider:
  • Serif fonts have a look that could be described as "classical"
  • Sans-serif fonts have a clean, modern look
  • Monospace fonts have equal width characters. Originating with typewriters, nowadays they're often used to display code

Of course, when choosing a font style, aesthetics can and should be a concern. However, you should also consider readability; for example, sans-serif fonts are often more readable at smaller sizes than serif fonts. On the other hand, serif fonts can be more readable at larger sizes, and at higher resolutions.

Web Safety

As mentioned before, web safety is a crucial consideration when choosing fonts for your website. If a font isn't supported by a user's browser or device, it will be replaced with a default font, which can lead to a disjointed and unprofessional appearance.

To ensure web safety, it's best to use widely supported fonts or to include fallback options in your CSS. For example:

font-family: 'CustomFont', Arial, sans-serif;

In this example, if 'CustomFont' isn't available, the browser will fall back to Arial, and if Arial isn't available, it will fall back to the default sans-serif font.

Custom Fonts

If you want to use a custom font that isn't widely supported, you can include it in your website using the @font-face rule in your CSS. This allows you to specify a font file that will be downloaded and used by the browser.

@font-face {
    font-family: 'CustomFont';
    src: url('CustomFont.ttf') format('ttf'),
    url('CustomFont.ttf') format('ttf');
 }

In the CSS above, the @font-face rule defines a custom font called 'CustomFont' and specifies the font files that should be used. The browser will download these files and use 'CustomFont' when it's specified in the font-family property.

Keep in mind that using custom fonts can increase the load time of your website. This is less of a concern nowadays, but it's still something to consider.

Getting Fonts

If we want customs fonts, we have to get them from somewhere. The most important thing to remember when you do this is that even on the internet, most things belong to someone. Chances are, if you want to use something, you'll have to pay for it, or at the very least have to adhere to a license agreement.

That being said, there are many resources available for finding and obtaining fonts, some of which are free. For example, Google has a large font library that is free to use for both personal and commercial projects. Still, make sure to read the corresponding license agreements and make sure you are complying with them.
